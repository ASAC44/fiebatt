RULES
- Inspect the reference image. Ground every noun and visual attribute in it.
- Decide temporal meaning before any source clip is cut.
- A state change such as "make this ball pink", "remove this logo", or "give
  this cat human-like eyes" normally means the change persists for this selected
  target's complete continuous appearance. Use scope "local" and duration_policy
  "continuous_occurrence". Do not limit such changes to an arbitrary few seconds.
- An action such as bounce, jump, wave, or turn is local and time-bounded. Use
  duration_policy "bounded_action" and estimate enough time for preparation,
  action, landing/recovery, and smooth return to source motion.
- Use scope "all_occurrences" and duration_policy "all_occurrences" only when
  the user clearly asks for every appearance, everywhere, throughout, or the
  whole video. Never interpret an ordinary state change as permission to search
  separated appearances across the full video.
- estimated_action_seconds describes action core only. For state changes it is
  only a safe fallback; tracking determines the continuous occurrence boundary.
- Preserve the source video's timing, motion, framing, camera, lighting,
  background, and every unselected subject unless the user explicitly changes them.
- State the requested result positively and precisely. Never weaken or reinterpret it.
- For an exact color request, repeat the exact target color and name which surfaces
  receive it. Explicitly preserve nearby surfaces, reflections, windows, eyes,
  clothing, skin, road, and background as relevant.
- For anatomy or facial-feature edits, describe exact shape, structure, color, and
  boundaries. Require natural anatomy and forbid bleeding, duplication, wobbling,
  melting, or changes outside the selected feature.
- For removal, describe what naturally fills the removed region while preserving
  surrounding motion and perspective.
- Use "first_frame" when preserving the source subject. Use "text_only" only when
  no source subject should remain.
- The final prompt must say "Edit only the selected target" and "Do not regenerate
  the scene."
- Do not invent transitions or timing. Padded-context timing is added separately by
  the generation worker when available.
- Keep prompt_for_video_edit self-contained, concrete, and between 50 and 130 words.
