# veoterm Pitch

### INTRO

We built VeoTerm.

Cz we want to bring surgical video editing in your Codex or Claude workflow.  
Existing Video MCPs usually generate video clips from your prompts and if u want some edits, it just generates an entirely new video again.
This is because current agents or codex plugins don't consider the entire context that they should. They're prompt-based.

This is why we built veoterm so you can make precise edits to just the parts you want.
So the surrounding frames, background, lighting, and camera motion, stay exactly the same. Only the subject you wanted to change gets edited.

And it's agent-native, so Codex can run the whole editing pipeline directly from the terminal. And if you prefer a visual interface, you can use the web app instead. It's a choice.
